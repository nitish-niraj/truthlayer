# test_assets/ — Trap & Regression Dataset (V2 Phase 5)

This directory holds the **trap and regression test fixtures** used to verify
that future changes to TruthLayer do not silently regress accuracy. The
contents are intentionally not in the repository by default — they are
generated by the build pipeline that ships with the model that produced the
expected verdicts.

## Layout

```
test_assets/
├── pdfs/           # PDF documents used as the PDF pipeline input
├── images/         # Image files used as the image pipeline input
├── trap_images/    # Hand-curated "trick" images (statistics, charts, infographics)
├── stress/         # Synthetic large inputs for stress tests
├── expected_results.json
└── README.md       # (this file)
```

## expected_results.json

The framework contract — a single JSON file that the regression suite loads
to assert a known verdict distribution for every fixture.

```json
{
  "only_true_claims.pdf":   { "verified": 5, "inaccurate": 0, "false": 0 },
  "only_false_claims.pdf":  { "verified": 0, "inaccurate": 0, "false": 5 },
  "bar_chart_ev_sales.png": { "verified": 1, "inaccurate": 0, "false": 0 },
  "watermark_inflation.jpg":{ "verified": 0, "inaccurate": 0, "false": 1 }
}
```

Top-level keys are filenames (relative to the subdirectory of the same
name). Values are summary counts. The regression test asserts that the
actual pipeline summary is **consistent with** the expected counts — it
tolerates drift within bounds (see `tests/test_trap_dataset.py` for the
exact threshold) so a flapping LLM does not produce false failures.

## How the test consumes this

```python
import json
from pathlib import Path

ASSETS_DIR = Path(__file__).resolve().parents[1] / "test_assets"
EXPECTED = json.loads((ASSETS_DIR / "expected_results.json").read_text())
```

If the asset set is empty, the regression test is skipped (it has nothing
to assert against). When you drop a fixture in, add a matching entry to
`expected_results.json` with the verdicts you observed at the time the
fixture was captured.

## Adding a new fixture

1. Drop the file in the matching subdirectory (`pdfs/`, `images/`,
   `trap_images/`, or `stress/`).
2. Run it through the live pipeline once. Record the observed
   `summary.{verified,inaccurate,false}` for that filename.
3. Append the entry to `expected_results.json`.
4. Re-run `pytest tests/test_trap_dataset.py -v` — it should pass against
   the recorded values.

## Note on accuracy drift

The verifiers use a non-deterministic LLM. A claim that was "verified" last
month may flip to "inaccurate" the next time the model version rolls. The
regression test therefore asserts that the **distance** between the
expected and observed distribution is within a small tolerance — not that
each claim matches exactly.
